Working with Files in Repl.it

Introduction

In this guide we are going to be talking about how you can be comfortable with using and manipulating files in Repl.it. To see previous topics about Repl.it, take a look here and if you are not in the mood, don’t worry as this guide can be followed independently.

This guide uses the Python programming language but you should be able to follow it if you are familiar with another programming. As always, we explain thoroughly each line of code so that you are never left hanging.

A computer file is a computer resource that is used to store data and there are many different types of files such as video files, picture files and text files. The data contained in files can then be used by the programs on that particular computer it resides on or through remote programs via networking.

Files are important to programs because they store the data permanently that allows the programs to run on your computer. Without the files, these programs would not be able to run as the data would not be available!

File Creation

Files can be created manually or programmatically. To create a file manually in Repl.it, refer to this guide.

Programmatically creating files in Repl.it

The alternative to creating files manually is to create them programmatically i.e. through the use of a computer program. As mentioned earlier, we are going to be using the Python Programming language in Repl.it to manipulate files.

File manipulation in Python happens in the following sequence:

  1. Open the file
  2. Perform the manipulation
  3. Close the file

As you can see, you first have to open the file before you can perform an operation. Luckily for us, python provides us a key function called open(), which as the name implies opens a file for us. The open() function takes two parameters:

  1. Filename - this is the name of the file.
  2. Mode - this is the manipulation you want to do on the file. This is an optional parameter.

To give an example, consider the following line of code:

file = open("examplefile.txt", "w")

It creates a file object called file which is assigned the results of the open function. The open function opens the file examplefile.text in the write mode which is denoted by w.

Other than the write mode shown above, there are other various ways to manipulate files. These ways are determined by the operations that can be done on the files and they are:

Operation Mode
Write w
Append a
Exclusive Creation x
Read r
Delete

Write Operation

As the name implies, the write operation records information to a file i.e. it adds new information to the specified file. In Repl.it, you can use the write to create a file programmatically. To demonstrate this, just create a new repl, choose python as the programming language and name it CreateFile as shown below.

Click on the green Create Repl button to create the repl.

It will open up in the main.py file where we will write our code. Notice on the files pane, there is just the main.py file and nothing else.

On Line 1 of the main.py file, enter the following python code:

file = open("examplefile.txt", "w")

This line of code means that open the examplefile.txt file in the write mode. If the file does not exist, as in our case, create it.

Press the run button, and Repl.it will create for you the examplefile.txt file as it does not exist. If it existed, it would seem like nothing has happened as there would be no visible change to the file.

Our file has been created, and if you open it up, you will see that it is empty.

Now let’s actually write something to our examplefile.txt . Click on the main.py file and add the following code.

myfile.write("Writing to file.")
myfile.close()

Remember our file is open in the write mode so the first line adds the words “Writing to file.” on the file. The second line myfile.close() signifies that we are done using the file. You cannot perform any operation on a closed file and you should always close your files as in some cases due to buffering, modifications made to the file might not show until you close the file.

Click the run button and then open the examplefile.txt file. You will see that it now contains the words “Writing to file.”. Also notice that there is nothing displayed in the output pane. This is because the result of our code affects the file and that’s where the modification happens. We will see the output pane in action when we read from the file.

It must be noted that if you open a file in the write mode by using w in the open() function, and then provide a write operation like we did with myfile.write("Writing to file"), you will essentially overwrite everything that is in the file.

To show this, just open the main.py file and change the second line from:

myfile.write("Writing to file.")

to:

myfile.write("File Overwritten.")

Then press the run button to run the repl and open up the examplefile.txt. You will see that it now contains the words File Overwritten., the words Writing to file. have been overwritten.

To maintain the current contents of the file, you have to open it in append a mode.

Append Operation

The append operation is represented by the a mode. It basically means open the file for writing and then add on the new information at the end of the file if it exists. If the file does not exist it, create it and then add the new information.

Currently, we have our file examplefile.txt which exists and has the contents “File Overwritten.” To append to the end of the file, i.e. add new information after File Overwritten open the main.py file and make the changes described below:

Change line 1 from:

myfile = open("examplefiles.txt", "w")

to:

myfile = open("examplefiles.txt", "a")

i,e. the w is changed to an a.

Change Line 2 from:

myfile.write("File overwritten.")

to

myfile.write("\nLine appended.")

Now instead of writing “File overwritten” we are going to write “Line appended.” to our file. And because this is an append operation, we expect our examplefile.txtto now have two lines:

File overwritten.
Line appended.

\n means new line and it moves the cursor to a new line. If we do not put it the words “Line appended” will be added on the first line after the full stop on File overwritten. So without the line break, the output will be as follow:

File overwritten.Line appended.

Exclusive Creation

The w and a modes are safety nets in that if the file exists, they will simply open it and if it doesn’t exist, they will create it. However, the x mode is the exclusive creation operation. If you use the x mode and the file exists, we will get an error. The x mode means create a file only if it doesn’t exist and then open it for writing.

To demonstrate this operation, open up the main.py file and change the first line from:

myfile = open("examplefiles.txt", "a")

to:

myfile = open("examplefiles.txt", "x")

This changes the mode from append to exclusive create.

Leave line 2 as is, i.e. Line 2 should still read:

myfile.write("\nLine appended.")

And then press the run button. Because our file examplefile.txt exists, we will get the following error:

You might need to use the x mode to ensure that you don’t accidentally overwrite or append to a file.

Reading Operation

Up to now, we have been talking about writing to files. Now let’s switch gears and move to reading files in Repl.it using python.

To read files you need to open the file in the read mode which is represented by r. After you open the file, you can then use the read() function to read the contents of the file.

To make this interesting we are going to create a file manually, add some text to it and then write the code that will read that text.

So just click on new repl, select python as the programming language and name it ReadingFiles.

When it opens up, click on the Add File icon to create a new file.

and then give our file a name. I am going to call it read_sample.txt.

Next up, add the following text to the file.

This is Line 1.
This is Line 2.
This is Line 3.
This is Line 4.
This is Line 5.
This is Line 6.
This is Line 7.
This is Line 8.
This is Line 9.
This is Line 10.

Our file in Repl.it should like the following:

Now click on the main.py file to open it. And then add the following code:

myfile = open("read_sample.txt", "r")
contents = myfile.read()
print(contents)
myfile.close()

Line 1 opens our file read_sample.txt in read mode as specified by the r.

Line 2 uses the read() function on the our file object which is in the myfile variable. It then assigns the result of the read operation to the contents variable.

Line 3 print out the contents. This way, we can see if our code actually read the file.

Line 4 close our file object.

Click on run to run the file and you should get the text in our file printed on the output pane.
